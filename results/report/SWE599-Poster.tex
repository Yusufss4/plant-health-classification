  \begin{alertblock}{The Problem}
    Pests and diseases destroy up to \textbf{40\% of global crops} every
    year, about \textbf{US\,\$220 billion} in losses. Early, on-site
    detection from leaf images, protects yield and cuts blanket
    pesticide use.

    \vspace{1ex}
    \textbf{Goal:} a self-contained classifier that labels a live camera
    frame as \textbf{Healthy}, \textbf{Diseased}, or \textbf{Background}
    — running entirely on a \$15, 512\,MB Raspberry Pi Zero 2 W with
    \emph{no} cloud, GPU, or NPU.
  \end{alertblock}

  \begin{block}{System Architecture}
    \centering
    \roundedimg[width=\colwidth, height=0.46\textheight, keepaspectratio]{system_architecture_v2_crop.pdf}

    \vspace{0.5ex}
    \raggedright
    \small Camera (or stored images) $\rightarrow$ on-device preprocess
    $\rightarrow$ ONNX Runtime $\rightarrow$ web portal streamed over the LAN.
  \end{block}

  \begin{block}{Key Design Choices}
    \begin{itemize}
      \item \textbf{3-class topology} — a built-in \emph{background} class
        rejects empty scenes with no extra detection model.
      \item \textbf{FP32 over INT8} — the Cortex-A53 lacks
        \texttt{SDOT}/\texttt{UDOT}, so INT8 falls back to slow software;
        FP32 uses mature NEON FMA instructions.
      \item \textbf{Native C++ runtime} — keeps memory far under the
        512\,MB ceiling; zero-allocation hot loop.
      \item \textbf{MobileNet V3-Small} — only \textbf{1.52\,M} parameters.
    \end{itemize}
  \end{block}

  \begin{exampleblock}{Conclusion}
    A native C++ end to end pipeline runs MobileNet V3 at
    \textbf{$\sim$8 img/s} on a Pi Zero 2 W (live demo: \textbf{inference latency $\sim$68\,ms},
    \textbf{encoding latency $\sim$31\,ms} per frame). The background class removes the need for a second
    model, and FP32 beats INT8 on this CPU. \emph{Future work:} fine-tune on
    deployment-camera captures and add field validation in live greenhouses.
  \end{exampleblock}
